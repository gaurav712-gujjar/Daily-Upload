## 🛠️ What I Built / Debugged

Been working on building a trading bot signal system that can predict stock prices using a moving average strategy. The goal is to create a system that can send buy and sell signals based on the intersection of two moving averages. I spent most of the day debugging and refining the logic to handle different types of moving averages and time frames.

```python
import pandas as pd
import numpy as np

def calculate_moving_averages(prices, short_window, long_window):
    ma_short = prices.rolling(window=short_window).mean()
    ma_long = prices.rolling(window=long_window).mean()
    return ma_short, ma_long

def generate_signals(prices, short_window, long_window):
    ma_short, ma_long = calculate_moving_averages(prices, short_window, long_window)
    signals = np.where(ma_short > ma_long, 1, 0)
    return signals

prices = pd.Series([10, 12, 15, 18, 20, 22, 25, 28, 30])
short_window = 3
long_window = 6
signals = generate_signals(prices, short_window, long_window)
print(signals)
```

Initially, I encountered issues with the rolling window calculation, which was causing the moving averages to be misaligned with the price data. I fixed this by using the `rolling` function from pandas, which provides a more accurate and efficient way to calculate moving averages. I also had to adjust the signal generation logic to account for the difference in the short and long window sizes.
