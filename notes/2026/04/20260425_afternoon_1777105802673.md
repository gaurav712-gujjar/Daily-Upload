## 🛠️ What I Built / Debugged

Been working on designing a MySQL schema with foreign keys for a simple e-commerce database, trying to ensure data consistency and reduce data redundancy. The goal is to create a robust database structure that can handle large amounts of customer, order, and product data. I've been focusing on establishing relationships between tables such as customers, orders, and products.

```
CREATE TABLE customers (
  customer_id INT PRIMARY KEY,
  name VARCHAR(255),
  email VARCHAR(255)
);

CREATE TABLE orders (
  order_id INT PRIMARY KEY,
  customer_id INT,
  order_date DATE,
  FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);

CREATE TABLE products (
  product_id INT PRIMARY KEY,
  product_name VARCHAR(255),
  price DECIMAL(10, 2)
);

CREATE TABLE order_items (
  order_id INT,
  product_id INT,
  quantity INT,
  FOREIGN KEY (order_id) REFERENCES orders(order_id),
  FOREIGN KEY (product_id) REFERENCES products(product_id)
);
```

Initially, I encountered issues with establishing the foreign key constraints, as I forgot to specify the referenced table and column, resulting in SQL errors. I fixed this by carefully reviewing the MySQL documentation and ensuring that my foreign key constraints were correctly defined. I also had to adjust the order in which I created the tables to avoid referencing tables that didn't exist yet.

I still need to populate the database with sample data and test my schema to ensure it's working as expected.
