## 💡 What I Learned Today

Today I learned about scikit-learn Pipeline with ColumnTransformer, which allows me to chain multiple data preprocessing steps together and apply them to specific columns of my dataset, making it easier to manage complex data workflows. This matters because it helps prevent data leakage and ensures that my machine learning models are trained on preprocessed data, which is essential for achieving good performance. I can also reuse and compose these pipelines to simplify my code and make it more modular.

```
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.linear_model import LogisticRegression
import pandas as pd

# create a sample dataset
data = pd.DataFrame({'A': [1, 2, 3], 'B': ['a', 'b', 'c'], 'C': [4, 5, 6]})

# define preprocessing steps
numeric_features = ['A', 'C']
categorical_features = ['B']
numeric_transformer = StandardScaler()
categorical_transformer = OneHotEncoder()

# create a ColumnTransformer
preprocessor = ColumnTransformer(
    transformers=[
        ('num', numeric_transformer, numeric_features),
        ('cat', categorical_transformer, categorical_features)
    ]
)

# create a pipeline
pipeline = Pipeline(steps=[
    ('preprocessor', preprocessor),
    ('logreg', LogisticRegression())
])
```

What surprised me is how easy it is to create complex data workflows using Pipeline and ColumnTransformer, but at the same time, it requires careful planning and attention to detail to ensure that the preprocessing steps are applied correctly. I also need to watch out for the order of the preprocessing steps, as it can significantly impact the performance of my model. Another thing to consider is that ColumnTransformer can be computationally expensive, especially when dealing with large datasets.

I want to explore next how to use Pipeline and ColumnTransformer with more advanced machine learning models, such as gradient boosting and neural networks.
