## 💡 What I Learned Today

Today I learned about scikit-learn Pipeline with ColumnTransformer, which is a game-changer for handling different data types in a dataset, such as combining numerical and categorical features. This matters because it simplifies the process of creating and managing complex workflows, making it easier to focus on model development and hyperparameter tuning. By using ColumnTransformer, I can preprocess my data in a more structured and efficient way.

```python
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import OneHotEncoder
import pandas as pd

# assume we have a dataframe with numerical and categorical columns
df = pd.DataFrame({'A': [1, 2, None], 'B': ['a', 'b', 'c']})

numerical_features = ['A']
categorical_features = ['B']

numerical_transformer = SimpleImputer(strategy='mean')
categorical_transformer = OneHotEncoder()

preprocessor = ColumnTransformer(
    transformers=[
        ('num', numerical_transformer, numerical_features),
        ('cat', categorical_transformer, categorical_features)
    ]
)

pipeline = Pipeline(steps=[('preprocessor', preprocessor)])
```

What surprised me is how flexible and customizable the ColumnTransformer is, allowing me to easily swap out different transformers and experiment with different preprocessing techniques. I also need to watch out for overfitting when using multiple transformers in a pipeline, as this can lead to decreased model performance on unseen data. It's also important to note that ColumnTransformer can be computationally expensive for large datasets.
