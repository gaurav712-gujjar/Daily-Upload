## 💡 What I Learned Today

Today I learned about scikit-learn Pipeline with ColumnTransformer, which allows me to chain multiple data processing steps together and apply different transformations to different columns of my dataset, making my data preprocessing code more modular and reusable. This is a game-changer for my machine learning workflow, as it enables me to experiment with different pipelines and compare their performance easily. By using ColumnTransformer, I can also handle categorical and numerical columns separately, which is often necessary in real-world datasets.

```
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import train_test_split
import pandas as pd

# assume X is a DataFrame with 'num_col' and 'cat_col' columns
num_transformer = StandardScaler()
cat_transformer = OneHotEncoder()
preprocessor = ColumnTransformer(
    transformers=[
        ('num', num_transformer, ['num_col']),
        ('cat', cat_transformer, ['cat_col'])
    ]
)
pipeline = Pipeline(steps=[('preprocessor', preprocessor), ('logreg', LogisticRegression())])
```

What surprised me is that ColumnTransformer automatically handles the output of each transformer and passes it to the next step in the pipeline, which is really convenient but also requires careful planning to ensure that the output of each step is compatible with the input of the next step. I also need to watch out for the fact that OneHotEncoder can create a large number of new columns if the categorical column has many unique values. 

I want to explore next how to use Pipeline and ColumnTransformer with more complex machine learning models, such as Random Forest or Gradient Boosting.
