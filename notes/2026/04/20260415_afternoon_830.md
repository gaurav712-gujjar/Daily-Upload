## 🛠️ What I Built / Debugged

Been working on designing a MySQL schema with foreign keys for a simple e-commerce database, trying to ensure data consistency and minimize redundancy. The goal is to create tables for customers, orders, and products, with relationships between them to facilitate efficient querying and data retrieval. This involves defining the structure of each table, including the data types and constraints for each column, as well as establishing the relationships between tables using foreign keys.

```
CREATE TABLE customers (
  customer_id INT AUTO_INCREMENT,
  name VARCHAR(255),
  email VARCHAR(255),
  PRIMARY KEY (customer_id)
);

CREATE TABLE orders (
  order_id INT AUTO_INCREMENT,
  customer_id INT,
  order_date DATE,
  PRIMARY KEY (order_id),
  FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);

CREATE TABLE products (
  product_id INT AUTO_INCREMENT,
  name VARCHAR(255),
  price DECIMAL(10, 2),
  PRIMARY KEY (product_id)
);

CREATE TABLE order_items (
  order_id INT,
  product_id INT,
  quantity INT,
  PRIMARY KEY (order_id, product_id),
  FOREIGN KEY (order_id) REFERENCES orders(order_id),
  FOREIGN KEY (product_id) REFERENCES products(product_id)
);
```

Initially, I ran into issues with the foreign key constraints, as I forgot to specify the ON DELETE and ON UPDATE actions, which led to errors when trying to insert or delete data. To fix this, I added the necessary clauses to the foreign key definitions, such as ON DELETE CASCADE and ON UPDATE RESTRICT. I also had to adjust the order in which I created the tables, as the foreign key constraints require the referenced tables to exist first.

What's left to do is to populate the tables with sample data and test the schema with various queries to ensure it meets the requirements of the e-commerce application.
