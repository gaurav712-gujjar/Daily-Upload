## 🛠️ What I Built / Debugged

Been working on building a trading bot signal system, specifically designing a MySQL schema to store historical stock data and trade signals. I needed to define tables for stocks, signals, and trades, with foreign keys to establish relationships between them. The goal is to efficiently query and analyze the data to improve the bot's performance.

```sql
CREATE TABLE stocks (
  id INT PRIMARY KEY,
  symbol VARCHAR(10) NOT NULL,
  name VARCHAR(100) NOT NULL
);

CREATE TABLE signals (
  id INT PRIMARY KEY,
  stock_id INT NOT NULL,
  signal_type VARCHAR(10) NOT NULL,
  timestamp DATETIME NOT NULL,
  FOREIGN KEY (stock_id) REFERENCES stocks(id)
);

CREATE TABLE trades (
  id INT PRIMARY KEY,
  signal_id INT NOT NULL,
  trade_type VARCHAR(10) NOT NULL,
  timestamp DATETIME NOT NULL,
  FOREIGN KEY (signal_id) REFERENCES signals(id)
);
```

Initially, I encountered issues with defining the foreign key constraints, as I was using the wrong syntax, but I fixed it by consulting the MySQL documentation and adjusting the CREATE TABLE statements. I also had to rework the schema to ensure data consistency and avoid potential deadlocks. After resolving these issues, the schema is now properly set up to store and relate the trading data.

I still need to write the Python code to interact with this MySQL schema and start populating it with historical data to begin testing the trading bot's signal generation and trade execution.
