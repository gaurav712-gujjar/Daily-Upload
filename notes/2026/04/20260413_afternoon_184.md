## 🛠️ What I Built / Debugged

Been working on writing a Flask REST API with error handling, specifically designing a robust endpoint to handle user requests and return relevant data. The goal is to create a system that can gracefully handle various types of errors, such as invalid input, database connection issues, and unexpected exceptions. I've focused on building a modular error handling mechanism that can be easily extended to different parts of the API.

```
from flask import Flask, jsonify
from functools import wraps

app = Flask(__name__)

def error_handler(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        try:
            return func(*args, **kwargs)
        except ValueError as e:
            return jsonify({'error': str(e)}), 400
        except Exception as e:
            return jsonify({'error': 'Internal Server Error'}), 500
    return wrapper

@app.route('/users', methods=['GET'])
@error_handler
def get_users():
    users = query_database()
    if not users:
        raise ValueError('No users found')
    return jsonify([user.to_dict() for user in users])
```

Initially, I struggled with implementing the error handling decorator, as it was not correctly catching and handling exceptions, but after reviewing the code, I realized that the issue was due to the incorrect placement of the decorator, which I then fixed by applying it directly to the route function. Additionally, I had to modify the decorator to handle different types of exceptions and return relevant error messages. 

I still need to test the API with various input scenarios to ensure the error handling mechanism is working as expected.
