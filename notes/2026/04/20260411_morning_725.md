## 💡 What I Learned Today

Today I learned about scikit-learn Pipeline with ColumnTransformer, which allows me to chain multiple data processing steps together and apply different transformations to specific columns of my dataset, making it a powerful tool for data preprocessing. This matters because it helps me avoid data leakage and improve model performance by ensuring that each step is applied in the correct order. I can also reuse the same pipeline across different models and datasets, making it easier to compare results and reproduce experiments.

```
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.linear_model import LogisticRegression
import pandas as pd

# assume we have a dataframe df with columns 'age', 'sex', 'country'
numeric_features = ['age']
categorical_features = ['sex', 'country']

transformer = ColumnTransformer(
    transformers=[
        ('num', StandardScaler(), numeric_features),
        ('cat', OneHotEncoder(), categorical_features)
    ]
)

pipeline = Pipeline([
    ('transform', transformer),
    ('model', LogisticRegression())
])

df = pd.DataFrame({
    'age': [25, 30, 35],
    'sex': ['male', 'female', 'male'],
    'country': ['USA', 'Canada', 'USA']
})
pipeline.fit(df.drop('target', axis=1), df['target'])
```

What surprised me is how easy it is to create complex pipelines with multiple steps and transformations, but also how important it is to carefully consider the order of operations to avoid data leakage or incorrect results. I also need to watch out for the computational cost of applying these transformations, especially when working with large datasets. 

I want to explore next how to use these pipelines with more advanced models, such as gradient boosting or neural networks, to see how they can be used to improve model performance and robustness.
