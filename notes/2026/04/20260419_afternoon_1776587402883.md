## 🛠️ What I Built / Debugged

Been working on building a trading bot signal system, specifically designing a MySQL schema with foreign keys to store and manage trading data. The goal is to create a robust database that can handle large amounts of data and provide efficient querying capabilities. I need to ensure that the schema is properly normalized to minimize data redundancy and improve data integrity.

```sql
CREATE TABLE symbols (
  id INT PRIMARY KEY,
  name VARCHAR(50) NOT NULL,
  exchange VARCHAR(20) NOT NULL
);

CREATE TABLE trading_data (
  id INT PRIMARY KEY,
  symbol_id INT,
  date DATE NOT NULL,
  open_price DECIMAL(10, 2) NOT NULL,
  high_price DECIMAL(10, 2) NOT NULL,
  low_price DECIMAL(10, 2) NOT NULL,
  close_price DECIMAL(10, 2) NOT NULL,
  FOREIGN KEY (symbol_id) REFERENCES symbols(id)
);

CREATE TABLE signals (
  id INT PRIMARY KEY,
  trading_data_id INT,
  signal_type VARCHAR(20) NOT NULL,
  generated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (trading_data_id) REFERENCES trading_data(id)
);
```

Initially, I encountered issues with creating the foreign key constraints, as I had incorrectly specified the data type of the referenced column. I fixed this by ensuring that the data types matched exactly, and also added NOT NULL constraints to the columns that will be used in the foreign key relationships. I then dropped and recreated the tables with the corrected schema.

I still need to write the application code that will interact with this database schema to store and retrieve trading data and signals.
