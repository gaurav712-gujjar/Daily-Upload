## 💡 What I Learned Today

Today I learned about scikit-learn Pipeline with ColumnTransformer, which is a powerful tool for building complex data processing workflows. I realized that it allows me to chain multiple data transformation steps together, making it easier to preprocess my data before feeding it into a machine learning model. This is particularly useful when dealing with datasets that have a mix of numerical and categorical features.

```python
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import OneHotEncoder, StandardScaler
from sklearn.linear_model import LogisticRegression
import pandas as pd

# assume we have a DataFrame df with columns 'age', 'city', and 'income'
numerical_features = ['age', 'income']
categorical_features = ['city']

preprocessor = ColumnTransformer(
    transformers=[
        ('num', StandardScaler(), numerical_features),
        ('cat', OneHotEncoder(), categorical_features)
    ]
)

pipeline = Pipeline([
    ('preprocessor', preprocessor),
    ('classifier', LogisticRegression())
])

# now we can use the pipeline to fit and predict on our data
```

What surprised me was how easy it was to create a pipeline with multiple transformation steps, and how it simplifies the process of experimenting with different preprocessing techniques. However, I also realized that it's important to carefully select the features to include in each transformation step, as this can significantly impact the performance of the model. One thing to watch out for is the potential for overfitting when using multiple transformation steps.

I want to explore next how to use the pipeline with more complex machine learning models, such as random forests or support vector machines.
