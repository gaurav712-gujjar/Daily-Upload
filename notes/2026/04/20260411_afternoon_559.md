## 🛠️ What I Built / Debugged

Been working on building a trading bot signal system, focusing on implementing a reliable method to generate buy and sell signals based on technical indicators. I needed to integrate moving averages and Relative Strength Index (RSI) calculations to create a robust signal generator. The goal is to create a modular system that can be easily extended with other indicators.

```python
import pandas as pd
import numpy as np

class SignalGenerator:
    def __init__(self, data):
        self.data = data

    def generate_signals(self):
        self.data['short_ma'] = self.data['close'].rolling(window=20).mean()
        self.data['long_ma'] = self.data['close'].rolling(window=50).mean()
        self.data['rsi'] = self.calculate_rsi(self.data['close'])
        self.data['signal'] = np.where((self.data['short_ma'] > self.data['long_ma']) & (self.data['rsi'] < 70), 1, 0)
        return self.data

    def calculate_rsi(self, series):
        delta = series.diff().dropna()
        up_days = delta.copy()
        up_days[delta <= 0] = 0
        down_days = delta.copy()
        down_days[delta > 0] = 0
        roll_up = up_days.rolling(window=14).mean()
        roll_down = down_days.rolling(window=14).mean().abs()
        rs = roll_up / roll_down
        rsi = 100.0 - (100.0 / (1.0 + rs))
        return rsi
```

When I first started, the RSI calculation was not accurate due to incorrect handling of the delta calculation, which led to incorrect up and down days. I fixed this by carefully adjusting the delta calculation to properly handle the series differences and then applying the rolling mean correctly. Initially, the signal generation was also missing a crucial condition that filtered out signals when the RSI was above 70, which I added to prevent overbuying.
