## 💡 What I Learned Today

Today I learned about scikit-learn Pipeline with ColumnTransformer, which is a powerful tool for machine learning workflows, allowing me to chain multiple data preprocessing steps together and apply them to specific columns of my dataset, making it easier to manage complex data pipelines. This matters because it helps prevent data leakage and makes my code more readable and maintainable. By using ColumnTransformer, I can apply different transformations to different columns, such as scaling numerical features and encoding categorical ones.

```python
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import train_test_split

# assuming X is my dataframe and y is my target variable
numeric_features = X.select_dtypes(include=['int64', 'float64']).columns
categorical_features = X.select_dtypes(include=['object']).columns

preprocessor = ColumnTransformer([
    ('num', StandardScaler(), numeric_features),
    ('cat', OneHotEncoder(), categorical_features)
])

pipe = Pipeline([
    ('preprocessor', preprocessor),
    ('classifier', LogisticRegression())
])
```

What surprised me is how easy it is to create complex pipelines with multiple steps, and how ColumnTransformer automatically handles the column selection for me, making it a huge time-saver. However, I need to watch out for the order of the steps in my pipeline, as it can significantly affect the performance of my model. I also need to be careful when using OneHotEncoder, as it can create a large number of new features if I have categorical variables with many unique values.
