## 💡 What I Learned Today

Today I learned about Flask blueprints and how they can help organize my code into smaller, more manageable pieces, which is super useful for larger applications where routing can get out of hand. I realized that using blueprints allows me to group related routes and functions together, making my code more modular and easier to understand. This matters because it makes it easier for me and others to maintain and extend the codebase.

```
from flask import Blueprint, Flask, render_template

app = Flask(__name__)
user_blueprint = Blueprint('user', __name__, template_folder='templates')

@user_blueprint.route('/profile')
def profile():
    return render_template('profile.html')

@user_blueprint.route('/settings')
def settings():
    return render_template('settings.html')

app.register_blueprint(user_blueprint)
```

What surprised me is how easy it is to create and register blueprints, and how they can be used to create a hierarchical structure for my routes. I also need to watch out for naming conflicts when registering multiple blueprints, as this can lead to unexpected behavior. I was also surprised by how much more readable my code becomes when using blueprints.

I want to explore using Flask blueprints in conjunction with other libraries and frameworks to see how they can be used to build even more complex and scalable applications.
