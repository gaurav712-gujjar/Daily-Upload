## 🛠️ What I Built / Debugged

Been working on designing a MySQL schema with foreign keys for a simple e-commerce database. I needed to create tables for customers, orders, and products, while ensuring data consistency and relationships between them. The goal was to allow efficient querying and minimize data redundancy.

```
CREATE TABLE customers (
  customer_id INT PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL
);

CREATE TABLE products (
  product_id INT PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  price DECIMAL(10, 2) NOT NULL
);

CREATE TABLE orders (
  order_id INT PRIMARY KEY,
  customer_id INT NOT NULL,
  product_id INT NOT NULL,
  order_date DATE NOT NULL,
  FOREIGN KEY (customer_id) REFERENCES customers(customer_id),
  FOREIGN KEY (product_id) REFERENCES products(product_id)
);
```

At first, I struggled with defining the relationships between the tables, and my initial schema had redundant data, but I realized I needed to use foreign keys to establish the connections. I fixed this by adding the FOREIGN KEY constraints to the orders table, which ensures data consistency and allows for efficient querying. I also had to adjust the data types for the price and order_date columns to better fit the requirements.

I still need to add indexes to the columns used in the WHERE and JOIN clauses to improve query performance.
