## 💡 What I Learned Today

Today I learned about scikit-learn Pipeline with ColumnTransformer, which allows me to chain multiple data processing steps together and apply different transformations to different columns of my data, making it a powerful tool for data preprocessing. This matters because it helps me to avoid data leakage and makes my code more modular and reusable. By using Pipeline and ColumnTransformer, I can easily experiment with different preprocessing techniques and evaluate their impact on my model's performance.

```python
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import train_test_split
import pandas as pd

# assume we have a dataframe df with numeric and categorical columns
numeric_features = ['age', 'income']
categorical_features = ['gender', 'education']

numeric_transformer = StandardScaler()
categorical_transformer = OneHotEncoder()

preprocessor = ColumnTransformer(
    transformers=[
        ('num', numeric_transformer, numeric_features),
        ('cat', categorical_transformer, categorical_features)
    ]
)

pipe = Pipeline(steps=[('preprocessor', preprocessor), ('classifier', LogisticRegression())])
X_train, X_test, y_train, y_test = train_test_split(df.drop('target', axis=1), df['target'], test_size=0.2)
pipe.fit(X_train, y_train)
```

What surprised me is how easy it is to create complex pipelines with multiple preprocessing steps, but it also requires careful consideration of the order of operations to avoid data leakage. I also need to watch out for the fact that ColumnTransformer can be computationally expensive for large datasets. Another thing to keep in mind is that Pipeline and ColumnTransformer are very flexible, but this flexibility can also make it harder to debug and understand what's going on.
